DC.gov Web Portal Analytics Report

Google Analytics Report: DC.gov Web Portal (2008 - March 2020)

Executive Summary

This report analyzes the top 100 page views per year on the DC.gov web portal from 2008 to March 2020. The focus is on three key pages: the DMV homepage, DC Health homepage, and a login/registration page. The analysis aims to improve and optimize these landing pages based on Google Analytics data.

1. DMV Homepage (/dmv.dc.gov/)

Year: 2020

Traffic Source: Google / Organic

Pageviews: 231,996

Unique Pageviews: 182,116

Avg. Time on Page: 0:00:55

Entrances: 127,768

Bounce Rate: 17%

Exit Rate: 25.9%

Analysis:

The DMV homepage receives significant organic traffic, indicating good SEO performance. However, the low average time on page (55 seconds) suggests that visitors may not be finding the information they need quickly.

Recommendations:

  • Redesign the page layout to improve user experience
  • Conduct user research to identify key information visitors are seeking
  • Implement a clear navigation structure
  • Consider adding a prominent search feature
  • Optimize page load speed to reduce bounce rate

2. DC Health Homepage (/dchealth.dc.gov/)

Year: 2020

Traffic Source: Direct / None

Pageviews: 222,549

Unique Pageviews: 202,766

Avg. Time on Page: 0:01:03

Entrances: 199,787

Bounce Rate: 81.6%

Exit Rate: 81.3%

Analysis:

The DC Health homepage has an alarmingly high bounce rate and exit rate, indicating that visitors are leaving the site without engaging with other pages. This could suggest technical issues or a mismatch between user expectations and page content.

Recommendations:

  • Consider adding internal links to relevant health-related content
  • Optimize page load speed and mobile responsiveness

3. Login/Registration Page (/cson.oag.dc.gov/LogReg.aspx?timeout=y)

Year: 2019

Traffic Source: Google / Organic

Pageviews: 231,368

Unique Pageviews: 29,132

Avg. Time on Page: 0:09:55

Entrances: 20,076

Bounce Rate: 50.4%

Exit Rate: 11.2%

Analysis:

The login/registration page shows a long average time on page, which could indicate user engagement but may also suggest difficulties with the form. The high bounce rate (50.4%) is concerning for a login page.

Recommendations:

  • Simplify the login/registration process
  • Implement a "Remember Me" option for returning users
  • Extend session length or add timeout warnings
  • Add clear error messages and validation for form fields

Overall Suggestions

This data showing a long average time on the page is promising, but the URL appears to be for the login page. This could mean that users are carefully filling out forms, which might indicate confusion or difficulty with the login/registration process. The bounce rate on this page is 50%, suggesting that many users leave without successfully logging in or registering. To address this, consider extending the session length or implementing a warning message before timeout. On the positive side, the low exit rate indicates that users who don't bounce continue to other pages on the site. Enhancing the user experience on this page is crucial, ensuring it is responsive on mobile devices. Adding a "remember me" option for returning users and improving site navigation to provide a clear path could also be beneficial.
